# チェックリスト（DoD）
## 仕様・品質
- [ ] 受け入れ条件（Given/When/Then）を満たす
- [ ] 例外系（通信失敗/権限拒否/空/ローディング/エラー）を考慮した
- [ ] 互換性（最低SDK/対象端末）に配慮した

## テスト
- [ ] 重要ロジックにUnit Testを追加/更新（or 対象外の理由を明記）
- [ ] UI導線が重要ならUIテスト/結合テストを追加（or 対象外の理由を明記）

## コード衛生
- [ ] Lint/静的解析/ビルドが通る（CIで確認）
- [ ] ログに個人情報・秘匿値を出していない（redaction済み）
- [ ] シークレットをコミットしていない

## 運用・観測（対象の場合）
- [ ] KPIに関係する導線は計測イベントを追加/確認した
- [ ] エラーはユーザー表示/再試行導線がある（握りつぶしていない）

## レビュー容易性
- [ ] PRはレビュー30分以内の規模（大きい場合は分割理由を明記）
- [ ] 変更点が読みやすい（命名/責務/コメント）
