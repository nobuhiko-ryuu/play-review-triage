# Definition of Ready / Done（DoR / DoD）

このドキュメントは、Backlog着手条件（DoR）と完了条件（DoD）を定義する。

---

## DoR（着手してよい条件）
以下が揃っていない場合、まずDiscoveryに戻して不足を埋める。

- [ ] ユーザーストーリーがある（誰の、何の課題を、どう解決するか）
- [ ] 受け入れ条件がある（Given/When/Then でYes/No判定できる）
- [ ] 画面が絡む場合：状態（Normal/Loading/Empty/Error/Permission）が定義されている
- [ ] 非機能の前提が書かれている（オフライン/性能/セキュリティ/対応OSなど、必要な範囲）
- [ ] “やらないこと”が明記されている（スコープ境界）
- [ ] 計測が必要か（必要ならイベント）が決まっている

---

## DoD（完了条件）
PRをマージしてよい条件。チェックはPRテンプレに従う。

最低限：
- [ ] 受け入れ条件を満たす
- [ ] CIがグリーン（Lint/静的解析/Unit/assemble）
- [ ] テストが追加/更新されている（or 対象外の理由が明記）
- [ ] 動作確認手順がPRに記載され、再現可能
- [ ] UI変更がある場合：スクショ/GIFがPRにある
- [ ] ログ・シークレット・例外系の扱いが適切

運用対象の場合：
- [ ] KPI導線の計測イベントが入っている
- [ ] エラー時のユーザー導線（再試行/案内）がある
